Introduction

This is still in progess, but it's for practicing design patterns and keeping data in sync while the user enters information in the form. It also updates the state on our own backend while you update this information. This application gathers all the information for a specific user and stores in angular's redux store. It's got all the basic api routes in place and a backend to play around and figure out different ways to send/update information either on update or submit.


This frontend is pretty straightforward and uses Angular store / Ngrx / Strongly typed forms / and hands free design patterns to capture and customize information The backend is built in express and uses the base routes with objection for handling a mySql database, and middleware error handling
